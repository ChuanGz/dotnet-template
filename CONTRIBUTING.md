# Contributing

## Purpose

Contributions must improve the template's correctness, clarity, maintainability, or practical value for teams building .NET services. This repository is not a discussion forum, technology debate space, personal blog, or collection of random tips.

## Accepted contributions

- Template implementation and defect fixes
- Tests for supported options and compatibility rules
- Documentation that enables a verifiable task or decision
- Focused proposals grounded in a concrete engineering problem

Every contribution must explain the problem, expected value, trade-offs, and when the change should not apply.

## Engineering standards

Keep changes scoped, testable, and consistent with the documented template contract. Prefer the smallest design that solves the stated problem. Explain new dependencies, abstractions, generated output, and compatibility impacts. Add or update tests for behavioral changes.

Feedback must be specific, respectful, evidence-based, and focused on the content. Technical disagreement should identify the relevant behavior, risk, trade-off, or maintainability impact.

## Rejected contributions

Contributions may be rejected when they contain:

- Generic opinions or personal preferences presented as standards
- Random tips without a repository use case
- Pattern-driven advice without a concrete problem
- Unnecessary complexity, abstraction, or dependencies
- Unrelated technology debates or promotional content
- Personal attacks or criticism of individuals, teams, companies, or styles
- Discussion that prioritizes winning an argument over improving the repository

Open an issue before material changes to the template contract or architecture. Keep pull requests focused and ensure relevant validation passes.
